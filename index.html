Claude Prompt Pack ‚Äî Weekly Scrumban Workflow

Paste before each prompt: "We are running Scrumban. We do not use fixed sprint goals. Work is task-flow based. Release schedule is every Monday and Thursday. Prioritize delivery reliability, risk visibility, and cross-team coordination."

Mon / Thu Release Readiness Check

Act as a senior Delivery Manager assistant. Using the task list below, prepare a release readiness report for the next deployment (Monday or Thursday):

  1. Tasks ready for release
  2. Tasks at risk
  3. Blockers / dependencies
  4. Rollback / prevention checklist
  5. Final go/no-go recommendation with reasons

Keep it concise and actionable.

Tue Mid-Week Flow Health Review

Analyze this Scrumban board snapshot and identify:

  1. WIP bottlenecks
  2. Aging tasks
  3. Throughput risks before next deployment
  4. Top 3 flow improvement actions for the next 3 working days
Wed Dependency & Escalation Planner

From the tasks below, detect cross-team dependencies and produce:

  1. Dependency map
  2. Risk level (high / medium / low)
  3. Escalation plan with owner and deadline
  4. Fallback options if dependency misses Monday/Thursday release
Fri Executive Status Update

Draft an executive weekly delivery update without using sprint terms. Structure:

  1. Delivered in last Monday/Thursday releases
  2. Currently in progress
  3. Top risks and mitigation
  4. Decisions needed from leadership
  5. Confidence level for next release windows
As needed Stakeholder Communication Rewriter

Rewrite this delivery message in 3 versions: neutral, assertive, executive. Keep facts unchanged. Include clear ask, owner, and due date. Avoid technical jargon unless necessary.

Wed Release Scope Trade-Off Advisor

Given limited capacity for the next Monday/Thursday deployment, prioritize tasks using business impact, risk, dependency, and effort. Return:

  1. Include now
  2. Defer
  3. Must-discuss trade-offs
  4. Impact of each option
As needed Incident & Stability Review

Based on recent incidents and deployment notes, generate:

  1. Recurring failure patterns
  2. Probable root causes
  3. Preventive actions before next release
  4. Owner-by-owner action list with target date
